Why LSD Usually Wins

  • Linear passes: LSD touches each element exactly once per digit place, so work is bounded by n * maxDigits.
  • Cache-friendly: Iterating a dense array keeps memory taps predictable, unlike MSD's recursive bucket shuffling.
  • Stable buckets: LSD maintains relative order on every pass, avoiding the merge bookkeeping MSD must redo.
  • Shallow control flow: No recursion eliminates call overhead and keeps the implementation short and branch-light.

The controls let you regenerate the dataset, single-step both flows, or auto-play the snapshot timeline. Watch the metrics: MSD re-examines digits inside each bucket, so bucket writes and time climb much faster than the iterative LSD version on the same input.
